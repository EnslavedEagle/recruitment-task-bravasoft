Zadanie rekrutacyjne - Bravasoft

Witaj w moim rozwiązaniu zadania rekrutacyjnego! W tym miejscu opowiem nieco o procesie, jaki miał miejsce podczas powstawania tej wspaniałej aplikacji.

  • Do stworzenia podstaw aplikacji użyłem Angular CLI.
  • Aplikacja jest podzielona na moduły i obsługuje lazy loading - zatem pobierają się z serwera dopiero wtedy, kiedy są potrzebne.
    Utworzone moduły to:
    • App, zarządzający całą resztą,
    • Home, który teraz widzisz,
    • Users, pozwalający zarządzać kolekcją użytkowników,
    • Movies, pozwalający zarządzać kolekcją filmów.
  • Pracę zacząłem od dodania dodatkowych ścieżek do pliku tsconfig.json co pozwoliło łatwo odwyływać się do części elementów, np. import {{'{'}} Costam } from '@interfaces/costam' zamiast import {{'{'}} Costam } from '../../../../../interfaces/costam'.
  • Boostrap 4 dodany został przez dodanie <link ...> do pliku index.html. Aby uzyskać dostęp do mixinów Bootstrapa, dodatkowo dorzuciłem pliki .scss z paczki boostrap-4-grid
  • Lista użytkowników stworzona została przy użyciu tabeli z zestawu @angular/material, która automatycznie wykonała paginację na podstawie przesłanego zestawu danych.
  • Tabele z kolekcji @angular/material niespecjalnie radzą sobie z responsywnością, ale dzięki temu, że ograniczyłem dane w niej wyświetlane - jakoś to wygląda. W przyszłości zapewne użyję raczej Bootstrapowych. :)
  • Widoki edycji korzystają z materialowych inputów (<input matInput> itd.)
  • Przy usuwaniu userów/filmów oczom użytkownika ukazuje się materialowy modal (MatDialog).
  • Do wykonywania żądań HTTP z użyciem klucza API w module Movies, skorzystałem z HTTPInterceptor, który pozwolił rozszerzyć obiekt request (w tym przypadku o Header Authorization: Bearer example-token).
  • Niniejsza aplikacja hostowana jest na Heroku z użyciem prostego serwera Express.js. Jeśli ładowała się długo, to przepraszam, Heroku usypia niepłatne instancje po 30 minutach nieaktywności. :)
